# ই-ভোটিং অ্যাপ ডেভেলপমেন্ট প্ল্যান

এই ডকুমেন্টটি ই-ভোটিং অ্যাপ্লিকেশন ডেভেলপমেন্টের পরিকল্পনা বর্ণনা করছে। বর্তমানে প্রোজেক্টের ফ্রন্টএন্ড কম্পোনেন্টগুলো প্রায় তৈরি হয়ে গেছে, তবে ব্যাকএন্ড এখনো **mock data** ব্যবহার করছে। এই পরিকল্পনার মূল লক্ষ্য হলো একটি শক্তিশালী ব্যাকএন্ড তৈরি করা, সেটিকে ফ্রন্টএন্ডের সাথে ইন্টিগ্রেট করা এবং প্রয়োজনীয় ফিচার যোগ করা যাতে অ্যাপ্লিকেশনটি সম্পূর্ণ কার্যকর হয়।

## 1. ডাটাবেজ সেটআপ

প্রথম ধাপে mock data এর পরিবর্তে একটি বাস্তব ডাটাবেজ ব্যবহার করতে হবে। এটি ডেটা স্থায়ীভাবে সংরক্ষণ করবে এবং অ্যাপটিকে আরও বাস্তবসম্মত করবে।

* **Task 1.1: ডাটাবেজ নির্বাচন:** এই প্রজেক্টের জন্য **mongodb** ইউজ করা হবে।
* **Task 1.2: ডাটাবেজ স্কিমা ডিজাইন:** বিদ্যমান mock data কাঠামোর উপর ভিত্তি করে ডাটাবেজ স্কিমা তৈরি করতে হবে। এতে থাকবে `users` as admin for maintaining application or super-admin, `candidates`, `voters`, `voting-schedule`, টেবিল।
* **Task 1.3: ডাটাবেজ কানেকশন:** Next.js অ্যাপ্লিকেশনে ডাটাবেজ কানেকশন সেটআপ করতে হবে। এজন্য এই অ্যাপ্লিকেশানে প্রিসমা ইউজ করা হবে।

## 2. API ডেভেলপমেন্ট

ডাটাবেজ সেটআপ হয়ে গেলে REST API তৈরি করতে হবে, যেটি ডেটার সাথে কাজ করবে।

* **Task 2.1: অথেন্টিকেশন API রুট:** ইউজার রেজিস্ট্রেশন, লগইন এবং লগআউটের জন্য API রুট তৈরি করতে হবে। ভোটারদের অথেনটিকেশন থাকবে না। শুধু অ্যাডমিন এবং সুপার অ্যাডমিন যারা ড্যাশবোর্ড কন্ট্রোল করবে তাদের জন্য অথেনটিকেশন থাকবে।
* **Task 2.2: অ্যাডমিন API রুট:** ভোটার, প্রার্থী এবং নির্বাচন শিডিউল ম্যানেজ করার জন্য অ্যাডমিন API রুট তৈরি করতে হবে। এর মধ্যে CRUD (Create, Read, Update, Delete) অপারেশন থাকবে।
* **Task 2.3: ভোটার API রুট:** নির্বাচন সম্পর্কিত ডেটা আনা, ভোট প্রদান এবং ফলাফল দেখার জন্য ভোটার API রুট তৈরি করতে হবে।
* **Task 2.4: Make Application Private:** অথেন্টিকেশন এবং অথরাইজেশন নিশ্চিত করতে Route এর মধ্যে রিডিরেক্ট ফিচার ইমপ্লিমেন্ট করতে হবে। মিডিলওয়ার থাকবে না।

## 3. অথেন্টিকেশন

একটি নিরাপদ অথেন্টিকেশন সিস্টেম তৈরি করতে হবে, যাতে ইউজারের ডেটা সুরক্ষিত থাকে এবং নির্দিষ্ট ফিচারে কেবল অনুমোদিত ইউজার প্রবেশ করতে পারে।

* **Task 3.1: NextAuth ইমপ্লিমেন্টেশন:** অথেন্টিকেশনের জন্য NextAuth v5@beta ব্যবহার করতে হবে। এবং সবকিছু এটার ওয়েতেই হ্যান্ডেল করা হবে।
* **Task 3.2: পাসওয়ার্ড সুরক্ষিত সংরক্ষণ:** ইউজারের পাসওয়ার্ড **hash এবং salt** করে ডাটাবেজে রাখতে হবে। এর জন্য `bcrypt` লাইব্রেরি ব্যবহার করা যাবে।

## 4. কম্পোনেন্ট ইন্টিগ্রেশন

বিদ্যমান React কম্পোনেন্টগুলোকে নতুন API এর সাথে যুক্ত করতে হবে যাতে সেগুলো কার্যকর হয়।

* **Task 4.1: অ্যাডমিন ড্যাশবোর্ড ইন্টিগ্রেশন:** অ্যাডমিন ড্যাশবোর্ড কম্পোনেন্টগুলো API এর সাথে যুক্ত করতে হবে, যাতে ভোটার, প্রার্থী এবং নির্বাচন ম্যানেজ করা যায়।
* **Task 4.2: ভোটার ফ্লো ইন্টিগ্রেশন:** ভোটার কম্পোনেন্টগুলো API এর সাথে যুক্ত করতে হবে, যাতে ভোটাররা নিরাপদে ভোট দিতে পারে।
* **Task 4.3: ফর্ম হ্যান্ডলিং:** ফর্ম হ্যান্ডলিং এবং ভ্যালিডেশনের জন্য `react-hook-form` এবং `zod` ব্যবহার করতে হবে।
